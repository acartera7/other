Lets continue on my project to make a MicSpammer using Qt and WASAPI. A software that will allow the user to browse their computer, load up audio from their computer and map them to the numpad, then at the press of one of the buttons on the numpad the audio will playback to the listener via a separate thread but also send the audio through a VB-Audio Cable so that it can be sent through to the mic and be heard by others in a game. Right now a lot of features are missing including proper selection of audio devices, playback and throughput into the mic and maybe eventual midi controls.

We currently have : AudioLoader.cpp AudioPlayer.cpp MicSpammerWindow.cpp FileBrowserWidget.cpp WasapiManager.cpp NumpadWidget.cpp and of course main.cpp. 

I added the feature to hear the audio file being played when the user browses through the file browser and double clicks a file. The file gets loaded on single click via Qt Multimedia decoder and then played on double click. A problem occurs however that it seems only a portion of the audio is loaded and then played, probably because the audio decoder buffers the audio and we need to loop it until the whole file is loaded. This also becomes a problem later when playing the audio as we need to tell the emerging thread to wait for the decoder to finish before playing, which also means we need to, if the file is big enough, create some sort of feedback to the user that the software is loading by putting a task bar or a loading symbol.
